## Sampled5 MVEE Prototype Path (Arbitrary-Camera Oriented)

Implemented a prototype projection/binner path that fits a conservative screen-space ellipse radius from 5 projected silhouette points using fixed-iteration MVEE math.

### Shader changes

- Updated `shaders/renderer/gaussian_pipeline.slang`:
  - Added helper math for sampled projection path:
    - 5-point silhouette generation in local unit sphere space
    - local-point to world transform
    - projection shim with optional radial distortion terms
    - fixed-iteration 5-point MVEE fit (3x3 inverse + 2x2 recovery)
    - shared tile-bin emission helper
  - Added new kernel:
    - `csProjectAndBinSampled5MVEE`
  - Kept existing `csProjectAndBin` as legacy path.

### Renderer changes

- Updated `src/renderer/gaussian_renderer.py`:
  - Added new constructor controls:
    - `projection_mode` (`legacy_axis_extent` / `sampled5_mvee`)
    - `sampled5_mvee_iters`
    - `sampled5_safety_scale`
    - `sampled5_radius_pad_px`
    - `sampled5_eps`
    - `proj_distortion_k1`, `proj_distortion_k2`
  - Loaded and dispatched new shader kernel for sampled mode.
  - Passed new sampled5 uniforms into project/bin dispatch.

### CPU reference changes

- Added `src/renderer/projection_sampled5_mvee_reference.py`:
  - CPU mirror of sampled5 silhouette + MVEE projection path.
  - Includes conservative fallback handling and status bits.
  - Exposes sampled screen points and fitted ellipse parameters for validation/tests.

### Tests
