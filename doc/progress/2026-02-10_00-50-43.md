# Universal X/Y scanline binning + renderer shader split

## Changes
- Implemented universal scanline span logic that works for both X- and Y-oriented scanlines.
- Scanline direction is now selected from the longest axis-aligned bbox side (`bboxExtent.x` vs `bboxExtent.y`).
- `emit_splat_bins` now supports both orientations while preserving two-pass (count/allocate, then write) behavior.
- Split renderer shader into modules and reduced per-file size to around <=250 LOC:
  - `gaussian_bindings.slang`
  - `gaussian_math_utils.slang`
  - `gaussian_mvee_utils.slang`
  - `gaussian_scanline_utils.slang`
  - `gaussian_binning_utils.slang`
  - `gaussian_project_stage.slang`
  - `gaussian_tile_range_stage.slang`
  - `gaussian_raster_stage.slang`
  - `gaussian_pipeline.slang` now only wraps entrypoints.
- Updated CPU reference binning (`reference_cpu.py`) to mirror universal X/Y scanline behavior and bbox-direction selection for parity tests.

## Validation
- `python -m pytest -q tests/test_renderer_pipeline.py` -> `5 passed`
- `python -m pytest -q` -> `11 passed`
