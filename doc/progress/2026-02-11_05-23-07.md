# Unify Raster Backward Shared Gradient Cache

## Summary
- Replaced four separate per-attribute `groupshared` gradient caches with one contiguous `groupshared` gradient-state array.
- Switched shared-cache access to explicit field offsets and a per-splat stride.
- Kept the two-stage accumulation flow unchanged: shared atomics first, global atomics flush second.

## Changes
- `shaders/renderer/gaussian_types.slang`
  - Added shared gradient-state layout constants:
    - `SPLAT_GRAD_STATE_POS_LOCAL_OFFSET`
    - `SPLAT_GRAD_STATE_INV_SCALE_OFFSET`
    - `SPLAT_GRAD_STATE_QUAT_OFFSET`
    - `SPLAT_GRAD_STATE_COLOR_ALPHA_OFFSET`
    - `SPLAT_GRAD_STATE_STRIDE`
  - Replaced four `gsGrad*` arrays with one `groupshared float gsGradState[RASTER_BATCH * SPLAT_GRAD_STATE_STRIDE]`.

- `shaders/renderer/gaussian_raster_stage.slang`
  - Replaced per-attribute shared-atomic helpers with a unified offset-based helper:
    - `atomic_add_grad4_shared(uint localIdx, uint fieldOffset, float4 grad)`
  - Added `grad_state_base(...)` and `load_grad4_shared(...)` helpers for indexed access.
  - Simplified cache clear to a single stride loop in `clear_batch_gradient_cache(...)`.
  - Updated `flush_batch_gradient_cache(...)` to read from `gsGradState` using field offsets.
  - Updated `RasterGaussian.accumulate_gradients_shared(...)` to use offset-based shared writes.
