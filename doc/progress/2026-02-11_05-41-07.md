# Camera Basis Matrix Refactor

## Summary
- Replaced per-axis camera uniforms (`g_camRight`, `g_camUp`, `g_camForward`) with one camera basis matrix uniform (`g_camBasis`).
- Switched both world-to-camera conversion and ray construction to matrix math.

## Changes
- `shaders/renderer/gaussian_types.slang`
  - Removed `g_camRight`, `g_camUp`, `g_camForward` uniforms.
  - Added `uniform float3x3 g_camBasis`.

- `shaders/renderer/gaussian_math_utils.slang`
  - `world_to_camera(v)` now uses `mul(g_camBasis, v)`.
  - Added `camera_to_world(v)` using `mul(transpose(g_camBasis), v)`.

- `shaders/renderer/gaussian_raster_stage.slang`
  - Replaced ray direction construction in forward and backward passes with:
    - `normalize(camera_to_world(float3(rayUV, 1.0)))`

- `src/renderer/camera.py`
  - `gpu_params(...)` now packs `(right, up, forward)` into a `3x3` matrix and uploads it as `spy.float3x3` via `g_camBasis`.
