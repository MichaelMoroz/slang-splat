# GPU-First Training/Debug Refactor

## Summary
- Removed CPU-heavy debug path and per-step image uploads.
- Separated main, training, and debug renderer responsibilities.
- Added per-parameter ADAM learning rates with lower default scale LR.

## Key Changes

### Trainer (`src/training/gaussian_trainer.py`)
- Reworked `AdamHyperParams`:
  - `position_lr`
  - `scale_lr`
  - `rotation_lr`
  - `color_lr`
  - `opacity_lr`
- Kept `scale_lr` lower by default (`2.5e-4`) than other parameter groups (`1e-3`).
- Preloads all dataset images to GPU textures at init:
  - Native resolution textures for debug comparisons.
  - Training-resolution textures for optimization.
- Removed per-step CPU image load/resize/upload.
- Added frame helpers:
  - `frame_count()`
  - `frame_size(...)`
  - `get_frame_target_texture(...)`
  - `make_frame_camera(...)`
- Added `update_hyperparams(...)` with flip-aware texture rebuild behavior.
- Optimization step now reads back only scalar loss; ADAM dispatch is submitted without extra synchronous wait.

### Training Shader (`shaders/renderer/gaussian_training_stage.slang`)
- Extended `AdamParams` to per-parameter learning rates.
- Updated fused ADAM kernel to apply dedicated LR vectors per parameter group.
- Added group-shared loss reduction path in `csComputeMSELossGrad` and one global atomic add per thread group.
- Added `csComposeAbsDiffDebug` kernel for on-GPU abs-difference visualization.

### Renderer (`src/renderer/gaussian_renderer.py`)
- Added `copy_scene_state_to(...)` for GPU buffer copies between renderers (no CPU scene readback).
- Extended `render_to_texture(...)` with `read_stats` flag to avoid per-frame stat readbacks when not needed.

### Viewer (`viewer.py`)
- Main renderer now remains full screen resolution.
- Added separate training/debug renderer lifecycle:
  - Training renderer for optimization resolution.
  - Debug renderer at selected image resolution.
- Added GPU scene synchronization from training renderer to main/debug renderers.
- Debug mode now:
  - Uses selected frame camera at native frame resolution.
  - Renders forward only.
  - Displays `Rendered` / `Target` / `Abs Diff` entirely on GPU.
  - Disables main render path while debug mode is enabled.
- Debug frame slider range is clamped to dataset frame count.
- Added separate UI LR controls for position/scale/rotation/color/opacity.

### CLI (`train.py`)
- Replaced single `--lr` with:
  - `--lr-pos`
  - `--lr-scale`
  - `--lr-rot`
  - `--lr-color`
  - `--lr-opacity`
- Scale LR default kept lower.
- Replaced `--no-target-flip-y` with explicit `--target-flip-y`.

## Validation
- Elevated test run:
  - `23 passed in 13.35s`
