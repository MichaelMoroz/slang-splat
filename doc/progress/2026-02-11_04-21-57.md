# Forward Raster Reuses Replay Abstraction

## Summary
- Refactored the default forward raster kernel to reuse `raster_forward_replay(...)` instead of keeping a separate duplicated batch loop.
- Preserved output composition and debug behavior while unifying splat traversal logic between forward and backward paths.

## Code Changes
- `shaders/renderer/gaussian_raster_stage.slang`
  - In `csRasterize`, removed the inline batch prefetch/intersection/blending loop.
  - Replaced that logic with:
    - `RasterState state = raster_forward_replay(start, end, pixelActive, rayDir, lane, processedCount);`
  - Kept debug processed-count output based on `processedCount`.
  - Kept final color and alpha output behavior, now using `state.accum` and `state.trans`.
