# 2026-02-17 18:53:13 - Fix prepass counter feedback for capacity growth

## Problem
- Prepass list/scanline capacity adaptation depended on counter readback feedback.
- Training path used `execute_prepass_for_current_scene(..., sync_counts=False)` and never consumed stats feedback, so `_pending_min_list_entries` was not reliably updated.
- Result: generated splat-tile entry counts could stay capacity-limited without renderer reallocating promptly in training.

## Changes
- `src/renderer/gaussian_renderer.py`
  - Added `_apply_counter_feedback(generated, capacity)` to centralize overflow bookkeeping and pending-capacity requests.
  - Reused this helper in:
    - `_update_delayed_counter_stats()` (render stats path)
    - `_execute_prepass(..., sync_counts=True)` (immediate/synchronous prepass path)
  - Updated `execute_prepass_for_current_scene(...)` to call:
    - `_ensure_work_buffers(self._current_scene.count, self._pending_min_list_entries)`
    - before dispatching prepass, so pending capacity growth requests are actually applied in this code path.
- `src/training/gaussian_trainer.py`
  - Training step now runs prepass with `sync_counts=True` to consume the 1-counter feedback each iteration and keep capacity adaptation active.
