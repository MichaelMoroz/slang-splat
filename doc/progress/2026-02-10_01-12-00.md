# Conservative scanline fix for missing splats

## Problem
- Approximate per-line conic interval fitting (sampling at 3 fixed positions per scanline) was not fully conservative.
- A geometric audit on `flowers.ply` found dropped intersecting tiles:
  - sampled splats: 250
  - splats with missing tiles: 98
  - missing tiles total: 152

## Changes
- Replaced approximate scanline interval solve in shader with exact conservative tile intersection testing per line:
  - Added conic box-min evaluation (`min_conic_over_tile_box`) in `gaussian_scanline_utils.slang`.
  - Scanline span now scans minor tiles and takes first/last intersecting tile, guaranteeing conservative coverage.
  - Circle fallback uses exact closest-point tile-vs-circle test.
- Mirrored same conservative span logic in `src/renderer/reference_cpu.py` for parity tests.

## Validation
- Geometric audit rerun after fix:
  - sampled splats: 250
  - splats with missing tiles: 0
  - missing tiles total: 0
- `python -m pytest -q tests/test_renderer_pipeline.py` -> `5 passed`
- `python -m pytest -q` -> `11 passed`
