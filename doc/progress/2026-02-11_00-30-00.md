# Simplify `compute_scanline_tile_span_universal`

**Date:** 2026-02-11

## Changes
- Replaced the 4-branch endpoint-hit case analysis (both hit, left-only, right-only, neither) plus exponential seed search plus linear fallback with a single unified flow.
- Removed `scanline_compute_span_linear_fallback` — no longer needed.

## New Algorithm
1. **Find seed tile**: Linear outward search from the ellipse center's projected tile coordinate. In practice hits on the first probe.
2. **Binary search boundaries**: Since an ellipse is convex, its intersection with any scanline is a contiguous span. Each half (left of seed, right of seed) is monotonic, so standard binary search finds the first and last hitting tile.

## Rationale
The convexity invariant makes all the special-casing redundant. The previous 4 endpoint cases, exponential probing, and linear fallback all reduced to finding *any* intersecting tile and then binary-searching both boundaries. ~60 lines → ~20 lines with identical behavior.
