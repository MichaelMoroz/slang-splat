# OOP-style shader refactor for scanline binning

## Changes
- Introduced renderer shader domain structs in `gaussian_types.slang`:
  - `Gaussian3D`, `Ellipsoid`, `Ellipse`, `ScanlineLoopConfig`, `ScanlineSpan`, `SplatBinRequest`.
- Added struct methods to encapsulate behavior:
  - `Ellipse.init(...)` (`[mutating]`)
  - `Ellipse.isVisible(...)`
  - `Ellipse.chooseScanAlongX(...)`
  - `ScanlineLoopConfig` helper methods for validation and tile-id composition.
- Refactored binning API to object-style call:
  - `emit_splat_bins(SplatBinRequest request)`
  - project stages now create and pass a request struct instead of long primitive argument lists.
- Unified scanline loop structure in binning:
  - counting and writing each use one shared scanline loop shape driven by `ScanlineLoopConfig` (no duplicated X/Y loop bodies).
- Updated project stage to use `Gaussian3D`/`Ellipsoid` objects for projection setup.
