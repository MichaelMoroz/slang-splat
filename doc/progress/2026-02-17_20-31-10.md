# 2026-02-17 20:31:10 - UI/Training controls refactor: Main tab, base LR model, remove Flip-Y

## Implemented
- `viewer.py`
  - Renamed `Scene` group to `Main`.
  - Moved to `Main` group:
    - scene load/reload controls
    - image dir selector
    - visual debug controls (`Visual Loss Debug`, `Debug View`, `Debug Frame`)
    - training control buttons
  - Renamed `Init Training Scene` button to `Reinitialize Gaussians`.
  - Placed `Camera` group directly below `Main` in UI construction order.
  - Replaced per-parameter LR inputs with:
    - `Base LR`
    - `LR Mul Position`
    - `LR Mul Scale`
    - `LR Mul Rotation`
    - `LR Mul Color`
    - `LR Mul Opacity`
  - Multiplier defaults set to `1.0` for all params (including scale).
  - Effective Adam LR now computed as `base_lr * multiplier` for each parameter group.
  - Removed `Flip Target Y` control and its data flow.

- `src/training/gaussian_trainer.py`
  - Removed `target_flip_y` from `TrainingHyperParams`.
  - Removed flip-cache state and flip-dependent dataset refresh logic.
  - Dataset texture upload now always uses native image orientation.

- `cli.py`
  - Replaced per-parameter LR args with base+multiplier args:
    - added `--lr-base`
    - added `--lr-mul-pos`, `--lr-mul-scale`, `--lr-mul-rot`, `--lr-mul-color`, `--lr-mul-opacity`
    - removed `--lr-pos`, `--lr-scale`, `--lr-rot`, `--lr-color`, `--lr-opacity`
  - Removed `--target-flip-y`.
  - Effective LR mapping now uses `base * multiplier` with clamping.
