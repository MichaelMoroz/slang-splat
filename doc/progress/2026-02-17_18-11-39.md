# 2026-02-17 18:11:39 - COLMAP init sampling cap fix

## What changed
- Updated `initialize_scene_from_colmap_points` in `src/scene/colmap_loader.py` so requested splat count is no longer clamped to COLMAP point count.
- Added replacement sampling when `max_gaussians > point_count`:
  - `chosen_count` now respects requested max (or all points when `max_gaussians <= 0`).
  - `rng.choice(..., replace=True)` is used only when requested count exceeds available points.
- Kept unique sampling behavior when requested count is within available points.

## Why
- Previous logic used `min(max_gaussians, point_count)` with `replace=False`, which hard-capped initialization to the sparse point cloud size (~54k in your dataset).
- Training/debug workflows need higher initial splat counts by reusing source points with different random noise/jitter.
