# Async GPU-driven prepass + raster loop optimization

## Implemented
- Reworked prepass scheduling to be GPU-driven from `g_ListCounter`:
  - Added indirect-args support for clamped element count and build-range dispatch in radix arg generation shaders.
  - Added `sort_key_values_from_count_buffer(...)` in `src/sort/radix_sort.py`.
  - Switched renderer prepass flow to project/bin + clear ranges + radix sort (indirect from counter) + build ranges (indirect).
  - Removed same-frame CPU count readback from render control flow.

- Added async delayed counter telemetry in `src/renderer/gaussian_renderer.py`:
  - Ring-buffered counter copy/readback.
  - Delayed stats fields: `stats_valid`, `stats_latency_frames`.
  - Overflow detection from delayed stats and deferred capacity growth request.

- Optimized raster inner loop in `shaders/renderer/gaussian_raster_stage.slang`:
  - Added cheap screen-space radius and conic reject checks before local-space ray evaluation.
  - Kept early transmittance termination and debug modes compatible.

- Updated shader interfaces:
  - `shaders/renderer/gaussian_types.slang`: prepass params buffer + sorted-count offset uniform + new groupshared staging data.
  - `shaders/renderer/gaussian_project_stage.slang`: tile-range build reads sorted count from GPU prepass params.

- Updated UI/tests/docs:
  - `viewer.py`: render stats display indicates delayed/warming states.
  - `tests/test_renderer_pipeline.py`: added delayed-stats semantics test.
  - `doc/Rendering.md`: documented GPU-driven indirect prepass and delayed stats behavior.
