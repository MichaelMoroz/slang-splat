# Analytic projection fallback fix for inflated tile counts

## Summary
- Fixed a large generated-tile overcount caused by projection fallback behavior in the analytic prepass path.
- Previously, unstable sampled-5 MVEE cases defaulted to `max_splat_radius_px`, which amplified tile writes on large scenes.
- Replaced that with an analytic fallback radius derived from projected ellipsoid scale and depth.

## Changes
- `shaders/renderer/gaussian_project_stage.slang`
  - Changed default/fallback projected radius initialization from hard max-radius to:
    - `fallbackRadius = g_focalPixels * max(ellipsoid.scale) / depth`
    - then clamped with existing safety/pad and max-radius controls.
- `src/renderer/projection_sampled5_mvee_reference.py`
  - Mirrored the same fallback formula for silhouette/project/fit fallback branches to keep CPU reference parity with shader behavior.
- `doc/Rendering.md`
  - Documented the analytic fallback-radius behavior in the project/bin stage.
