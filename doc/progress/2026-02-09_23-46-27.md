# Ellipse outline debug mode in rasterizer

## Summary
Added a rasterizer debug overlay mode that draws estimated screen-space ellipse outlines for projected splats.

## Shader changes
- Added `g_ScreenEllipseConic` (`float4` per splat) storing `(a00, a01, a11, valid)` for implicit ellipse form.
- Legacy projection kernel now writes circle-equivalent conic from radius.
- Sampled5 MVEE projection kernel now writes fitted conic from MVEE output (scaled to match conservative radius inflation).
- `fit_mvee_5pt_fixed_iters` now outputs conic coefficients in addition to center/radius.
- Raster kernel now supports optional outline overlay via uniforms:
  - `g_debugShowEllipses`
  - `g_debugEllipseThicknessPx`
  - `g_debugEllipseColor`

## Renderer changes
- Added work buffer allocation/binding for `screen_ellipse_conic`.
- Added renderer options:
  - `debug_show_ellipses`
  - `debug_ellipse_thickness_px`
  - `debug_ellipse_color`
- Bound debug uniforms during raster dispatch.
- Exposed `screen_ellipse_conic` through `debug_pipeline_data`.

## Viewer changes
- Added `Debug Ellipse Outlines` checkbox in Render Params.
- Checkbox updates renderer debug flag at runtime.
- Preserved debug flag when renderer is recreated.

## Validation
- `pytest -q` passed: `11 passed`.
