# Backprop Quaternion/Scale Chain Fix

## Scope
- Kept forward rendering path unchanged.
- Fixed missing gradient-chain terms on optimizer side in fused ADAM shader.

## Problem
- `grad_splat_pos_local` was only used to update position.
- But `roLocal` depends on:
  - position
  - quaternion
  - inverse-scale
- Missing chain terms caused incomplete backprop for orientation (and part of scale).

## Changes
- `shaders/renderer/gaussian_training_stage.slang`
  - Added analytic helper:
    - `quat_grad_from_rotate(float3 v, float4 q, float3 gradOut)`
  - In `csAdamStepFused`:
    - Added quaternion gradient contribution from `roLocal` dependency.
    - Added inverse-scale gradient contribution from `roLocal` dependency.
    - Kept existing quaternion normalization + canonical sign handling (`w >= 0`) after update.

- `tests/test_training_kernels.py`
  - Added:
    - `test_ro_local_grad_contributes_to_quaternion_update`
  - Verifies quaternion updates occur when only `grad_splat_pos_local` is injected.
  - Verifies resulting quaternions stay finite and approximately unit-length.

## Validation

  - `pytest -q tests/test_training_kernels.py` -> passed
  - `pytest -q tests/test_training_cli_smoke.py` -> passed
