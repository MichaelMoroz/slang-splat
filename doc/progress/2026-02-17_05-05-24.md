# Debug View Alignment Fixes

## Scope
- Fixed COLMAP debug-view rendering flow so it no longer uses screen resolution.
- Fixed remaining target-orientation mismatch default in viewer training/debug controls.

## Changes
- `viewer.py`
  - Added `_reset_loss_debug_state()` to fully reset debug renderer/texture/cache state.
  - Added `_sync_render_params_to_renderer(...)` to keep render settings consistent between renderers.
  - Added `_ensure_loss_debug_renderer(width, height)` so debug rendering runs at selected frame resolution.
  - Updated debug target loading to `_load_debug_target_rgba(frame, width, height)` and keyed cache by frame path + resolution + flip flag.
  - Updated debug render path to:
    - Render selected COLMAP frame camera with the debug renderer.
    - Sync scene from main renderer GPU (`read_scene_from_gpu`) when trainer step changes.
    - Display `Rendered` / `Target` / `Abs Diff` from the selected frame path and camera.
  - Reset debug resources on resize, scene loads, COLMAP loads, and training-scene initialization.
  - Changed `Flip Target Y` default from `True` to `False` to avoid default vertical inversion in debug comparison.

- `src/renderer/gaussian_renderer.py`
  - Added `read_scene_from_gpu()` to read positions/scales/rotations/color+alpha from renderer buffers and reconstruct a `GaussianScene` for debug renderer sync.

## Validation
- Parsed `viewer.py` with Python AST to confirm syntax validity.
- Slangpy/e2e tests were not completed in this iteration.
