# COLMAP Training Pipeline + Fused ADAM

## Summary
- Implemented a basic COLMAP-driven 3DGS training pipeline with viewer and CLI integration.
- Added backprop-driven optimization with a separate fused ADAM kernel (one thread per Gaussian).
- Extended camera/shader projection to anisotropic intrinsics (`fx/fy/cx/cy`) and principal point.

## Code Changes
- Added COLMAP ingestion and scene initialization:
  - `src/scene/colmap_loader.py`
  - `src/scene/__init__.py` exports updated
- Extended camera intrinsics/extrinsics support:
  - `src/renderer/camera.py`
  - supports explicit intrinsics + basis override + `Camera.from_colmap(...)`
- Updated renderer shader camera contract:
  - `shaders/renderer/gaussian_types.slang` (`focalPixels: float2`, `principalPoint: float2`)
  - `shaders/renderer/gaussian_math_utils.slang`
  - `shaders/renderer/gaussian_project_stage.slang`
  - `shaders/renderer/gaussian_raster_stage.slang`
- Added training shader stage:
  - `shaders/renderer/gaussian_training_stage.slang`
  - kernels:
    - `csClearLossAndGradTex`
    - `csComputeMSELossGrad`
    - `csAdamStepFused`
- Added trainer runtime:
  - `src/training/gaussian_trainer.py`
  - `src/training/__init__.py`
- Added dedicated CLI training entrypoint:
  - `train.py`
- Viewer integration updates:
  - `viewer.py`
  - preserved PLY flow + added COLMAP load, training init, training controls, per-frame step, loss display
- Renderer integration helpers and writable scene buffers:
  - `src/renderer/gaussian_renderer.py`
- CPU reference intrinsics alignment:
  - `src/renderer/reference_cpu.py`
  - `src/renderer/projection_sampled5_mvee_reference.py`
