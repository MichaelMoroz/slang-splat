# MVEE default + ellipse-aware scanline binning

## Changes
- Default MVEE safety margin was changed to `1.0` in `src/renderer/gaussian_renderer.py` (`sampled5_safety_scale` default constructor value).
- `emit_splat_bins` in `shaders/renderer/gaussian_pipeline.slang` was rewritten from box-area emission to a two-pass scanline emitter:
  - pass 1 counts scanline tile spans and reserves entries with a single `InterlockedAdd(g_ListCounter[0], totalCount, baseIndex)`.
  - pass 2 recomputes spans and writes keys/values linearly.
- Added ellipse scanline helpers in shader:
  - `solve_ellipse_scanline_x_interval`
  - `compute_scanline_tile_span`
- Legacy projection path now passes its conic (`legacyConic`) to `emit_splat_bins` so binning can use ellipse-aware spans.
- Added matching CPU-reference scanline binning logic and legacy conic generation in `src/renderer/reference_cpu.py` to keep CPU/GPU parity checks valid.

## Stability note
- Enabling sampled5 MVEE conic scanline binning in prepass caused a GPU device-loss fault in tests on this machine.
- To keep runtime stable, sampled5 currently passes `0.0` conic to `emit_splat_bins`, which uses radius fallback for that path.
