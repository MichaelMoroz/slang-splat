# Debug processed-count visualization

- Added a new raster debug toggle to visualize per-pixel gaussian processing load.

## Renderer/API changes
- `src/renderer/gaussian_renderer.py`
  - Added constructor/runtime flag: `debug_show_processed_count` (default `False`).
  - Bound new shader uniform `g_debugShowProcessedCount` in raster dispatch vars.

## Shader changes
- `shaders/renderer/gaussian_types.slang`
  - Added uniform: `uint g_debugShowProcessedCount`.

- `shaders/renderer/gaussian_raster_stage.slang`
  - Added per-pixel `processedCount` accumulation inside the splat loop.
  - Added debug output path:
    - outputs grayscale, log-normalized count map when `g_debugShowProcessedCount != 0`.
    - empty-tile pixels output black in this mode.

## Viewer/UI changes
- `viewer.py`
  - Added UI checkbox: `Debug Processed Count`.
  - Propagated checkbox value through renderer recreation and per-frame parameter updates.

## Tests
- `tests/test_renderer_pipeline.py`
  - Added `test_debug_processed_count_render_smoke`.

## Validation
- `python -m pytest -q` -> `12 passed`.
