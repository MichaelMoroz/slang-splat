# Vulkan-Only Backend and Native Float Atomic Raster Grads

## Summary
- Removed D3D12 runtime selection paths and made Vulkan the only supported backend path in app/test entrypoints.
- Simplified raster gradient accumulation to direct native float `InterlockedAdd` with no macro-based fallback path.

## Code Changes
- `src/common.py`
  - `device_type_from_name()` now accepts only Vulkan aliases.
  - `create_default_device()` now defaults to `spy.DeviceType.vulkan`.
  - Removed backend-dependent shader define injection.
- `render.py`
  - Removed CLI `--device`; renderer now always creates the default Vulkan device.
- `viewer.py`
  - Removed CLI `--device`; viewer now always creates the default Vulkan device.
- `tests/conftest.py`
  - Removed backend env-var selection and force Vulkan device creation in test fixture.
- `shaders/renderer/gaussian_types.slang`
  - Removed conditional grad buffer declarations; grad buffers are now always `RWStructuredBuffer<float>`.
- `shaders/renderer/gaussian_raster_stage.slang`
  - Removed all `SLANG_SPLAT_USE_NATIVE_FLOAT_ATOMICS` branches.
  - Kept only direct float `InterlockedAdd` accumulation in custom load derivative paths.
  - `csClearRasterGrads` now clears float grad buffers only.
- `README.md`
  - Updated runtime docs to Vulkan-only usage and removed backend-selection examples.
