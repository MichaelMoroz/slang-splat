# Raster Backward Uses Cached Forward State from `csRasterize`

## Summary
- Removed forward replay from `csRasterizeBackward`.
- Reused `csRasterize` as the forward state producer by caching terminal raster state and processed-step count per pixel.

## Code Changes
- `shaders/renderer/gaussian_types.slang`
  - Added per-pixel forward cache bindings:
    - `g_RasterForwardState : RWTexture2D<float4>`
    - `g_RasterProcessedCount : RWTexture2D<uint>`
- `shaders/renderer/gaussian_raster_stage.slang`
  - `csRasterize` now writes:
    - `float4(state.accum, state.trans)` to `g_RasterForwardState`
    - `processedCount` to `g_RasterProcessedCount`
  - `csRasterizeBackward` now reads cached forward state/count and no longer calls `raster_forward_replay`.
  - Reverse traversal still uses the same batch/prefetch path and `processedCount`-bounded prefix.
- `src/renderer/gaussian_renderer.py`
  - Added texture resources:
    - `_raster_forward_state_texture` (`rgba32_float`)
    - `_raster_processed_count_texture` (`r32_uint`)
  - Allocated and validated in `_ensure_work_buffers`.
  - Bound in both `_rasterize(...)` and `_rasterize_backward(...)`.
